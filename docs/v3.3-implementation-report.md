# v3.3 Implementation Report

v3.3 adds Enterprise Crypto Discovery to the v3.2 control-plane runtime without
changing the canonical service schema or the compatibility TLS policy
`X25519MLKEM768:X25519`.

## Delivered

- Language-aware source API scanning for C/C++, Java, Rust, Go, Python and Shell.
- Magic-based ELF, PE, Mach-O, archive and JAR detection, including extensionless executables.
- Read-only dynamic dependency/imported symbol inspection with bounded string fallback.
- Optional Linux process-to-crypto-library correlation through `/proc`.
- Schema v3 artifacts and runtime process records while retaining v2 compatibility fields.
- JSON and CSV export plus SQLite persistence for artifacts and processes.
- Risk-engine correlation for high-confidence source/binary/runtime usage.
- A 15-case deterministic enterprise scanner experiment integrated into the full experiment.

## Security boundary

The scanner never invokes a scanned executable, never loads a JAR/class, never
persists private-key bytes and skips symlinks. File, archive, process and count
limits constrain enterprise scans. Production scans still require explicit
authorization and an organization-defined data retention policy.

## Deferred

Decompilation, debug-info ingestion, container registry APIs, CodeQL/Semgrep/CBOM
imports, Kubernetes workload discovery and HSM/KMS inventory are not included in
this version. The output model is designed so those collectors can be added later.
