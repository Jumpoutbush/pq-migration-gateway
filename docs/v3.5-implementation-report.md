# v3.5 Implementation Report

v3.5 turns the framework's existing API and metrics into a customer-operable
single-node enterprise pilot. It preserves the deterministic v3.4 experiment
environment while introducing an independent runtime without demo backends.

## Delivered

- Idempotent `enterprise-init` generation of secrets, authorized scan mounts,
  pilot PKI, runtime directories and canonical service configuration.
- Interactive and automatable `pqctl onboard` service creation.
- Hardened host-network Enterprise Compose for Gateway, Manager API and metrics.
- Signed first release, explicit apply/history/rollback and operating targets.
- One-command authorized scan and asset listing targets.
- Prometheus and Grafana observability profile with pre-provisioned dashboard.
- Control-plane metrics for assets, assessments, scans, plans and migration state.
- Alert rules for Agent health, fallback, handshake and upstream TLS failures.
- Enterprise quickstart covering onboarding, cutover, observation and rollback.

## Compatibility

The canonical schema, adapters, scan-to-migration REST API, v3.4 experiments and
legacy v3 normalization remain supported. Demo and enterprise runtime state use
different directories and Compose projects.

## Remaining production work

The profile is a single-node pilot, not a complete clustered appliance. It does
not automate DNS/LB changes, issue enterprise certificates, configure
Alertmanager, provide HA state consensus, or prove application-specific
protocol behavior and production capacity.
