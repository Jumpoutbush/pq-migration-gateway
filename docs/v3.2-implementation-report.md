# v3.2 Implementation Report

## Outcome

v3.2 upgrades the Gateway Framework Core into a runnable single-node Control Plane Runtime while preserving the v3.1 service schema, adapter contract, data-plane renderer and v3 flat-config compatibility.

## Delivered

- Persistent Service and Policy resources with revision, source config version, operator and audit metadata.
- Explicit release state machine and append-only status history.
- Failed validation retention without publishing invalid desired state.
- Expanded REST API for resource CRUD, release detail/publication/rollback, Agent status, migration history, audit and metrics.
- Agent identity, current/desired version, heartbeat, health, reload result and stale detection.
- Automatic rollback with stage-specific failure history and counters.
- Prometheus control-plane and runtime TLS metrics.
- Extended CLI for resources, Agents, metrics and release inspection.
- Compose wiring for a shared metrics database and stable Agent identity.
- Idempotent `scripts/init_system.sh` bootstrap flow exposed through `make init`.
- Twenty-two offline regression tests covering v3, v3.1 and v3.2 behavior.

## Compatibility

- `schema_version: 4.0` remains unchanged.
- v3 flat documents continue to normalize in memory.
- Existing protocol adapters and all ten demo services remain unchanged.
- TLS 1.3 and the default `X25519MLKEM768:X25519` compatibility policy remain unchanged.
- Rollback still creates a new version instead of rewriting history.

## Deferred

- PostgreSQL control-plane storage and multi-node release coordination.
- Rolling or percentage deployment across multiple Gateway instances.
- Certificate and CA bundle lifecycle management.
- Vault, HSM/KMS and PKCS#11 providers.
- Web UI and Kubernetes Operator.
- Full production authorization/RBAC and external identity provider integration.

These are intentionally left for v3.3/v4.0 so v3.2 remains a coherent, testable control-plane runtime rather than an incomplete cluster platform.
