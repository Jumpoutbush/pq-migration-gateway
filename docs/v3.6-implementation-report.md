# v3.6 Implementation Report

v3.6 closes the gap between implemented control-plane primitives and the
customer-facing interface. Once deployed, day-2 operations no longer require a
local shell, Make target or direct SQLite access.

## Delivered

- OpenAPI 3.1 contract at `/openapi.json`, including reusable service, release,
  enterprise-scan and migration request schemas.
- Capability discovery at `/v1/capabilities`.
- Aggregated runtime/control status at `/v1/status`.
- Customer-facing `/v1/releases` lifecycle aliases with backward-compatible
  `/v1/configs` routes.
- One-call service validation and signed publication through `/v1/onboarding`
  and `/v1/services/{id}/publish`.
- REST-only Python client library and `pqapi` command wrapper.
- API-backed enterprise apply, history, rollback, scan, asset and audit targets.
- 14-case deterministic API-first experiment plus unit coverage.
- Dockerfile warning fix and actionable host scan-root errors.
- Loopback-by-default API binding with an explicit protected-proxy deployment
  option for remote enterprise automation.

## Boundary

Container startup, host mounts, firewall/DNS changes and production private-key
provisioning remain infrastructure operations. This is intentional: an API
running inside the control plane must not grant itself arbitrary host access.
